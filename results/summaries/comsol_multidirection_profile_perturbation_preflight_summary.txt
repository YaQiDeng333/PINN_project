COMSOL multi-direction profile perturbation preflight summary

Stage: 20.64 multi-direction excitation profile perturbation oracle ordering feasibility

1. 是否值得执行 20.64
Yes. Stage 20.61 showed single-height Bz oracle residual is near random for profile quality ordering, Stage 20.62 showed multi-height Bz does not improve ordering, and Stage 20.63 showed same-direction Bx/By/Bz does not improve ordering. These results do not test real excitation-direction changes.

2. 是否已有结果否定 multi-direction excitation route
No. Existing results negate the current same-direction observation family and current residual objective, but they do not falsify multi-direction excitation because the COMSOL source direction was not changed.

3. 是否需要 COMSOL
Yes. direction_90 and optional direction_45 must be generated by real COMSOL forward solves with changed ExternalCurrentDensity.Je. Rotating arrays, masks, or signals is forbidden.

4. COMSOL 是否支持改变磁化/激励方向
Static code inspection supports it. COMSOL script generate_mfl_polygon_pilot_v5.py creates MagneticFields ExternalCurrentDensity and sets source.set("Je", config["source"]["Je"]). The default base_config uses Je=["0", "1e6[A/m^2]", "0"]. Stage 20.64 must still run a hard direction probe to verify the field response changes, not just that the Je string is written.

5. 实际可用的 direction list
Planned target list:
- direction_0: default +Y, Je=["0", "1e6[A/m^2]", "0"]
- direction_45: equal XY, Je=["7.071067811865475e5[A/m^2]", "7.071067811865475e5[A/m^2]", "0"]
- direction_90: orthogonal +X, Je=["1e6[A/m^2]", "0", "0"]
direction_90 is mandatory. direction_45 is optional and may be dropped if probe fails.

6. direction_90 是否可用
Static feasibility: likely yes. Execution gate: direction_90 must produce finite fields and a no-defect / defect response that differs materially from direction_0. If response is approximately identical, implementation is failed and generation stops.

7. direction_45 是否可用
Static feasibility: likely yes. It is optional; if probe fails while direction_0 and direction_90 pass, continue with partial direction support and record the failure.

8. 是否支持每个 direction 下导出 Bx/By/Bz
Stage 20.63 confirmed expressions [mf.Bx, mf.By, mf.Bz] are stable for default direction. Stage 20.64 will first try three-axis export for all directions. If nonzero-direction Bx/By are unstable but Bz is stable, all directions are consistently audited as Bz-only.

9. 实际使用的 field expressions
Preferred expressions: [mf.Bx, mf.By, mf.Bz]. Fallback expressions, only if nonzero-direction three-axis export fails: [mf.Bz].

10. 是否只做 oracle residual ordering
Yes. No forward surrogate training, inverse model training, profile refinement, dense baseline, or baseline update is allowed.

11. 唯一推荐执行方案
Reuse 20.63 profile rows/geometries for same-geometry comparison; make direction_0 the same-pack baseline by reusing exact 20.63 default-direction rows; generate direction_90 and optional direction_45 with real COMSOL source-direction changes; audit same-pack direction_0 Bz, direction_0 all-axis, direction_90 Bz, direction_90 all-axis, direction_45 if present, and multi-direction normalized residuals.

12. 必须避免的问题
- Do not fake multi-direction by rotating arrays, signals, masks, or coordinates.
- Do not treat a changed Je string as sufficient without field-response verification.
- Do not use 20.63 historical values as the main comparison gate.
- Do not train a surrogate or run refinement in Stage 20.64.
- Do not submit data, NPZ, .mph, raw CSV, checkpoint, preview PNG, notes, __pycache__, or baseline docs.

15. stop conditions
- direction_90 cannot be implemented by a real COMSOL source-direction change.
- direction_90 fields are non-finite or too similar to direction_0.
- split/type/variant/profile row minimum coverage fails.
- delta_B != B_defect - B_no_defect checks fail.
- No true_reference row is available per base for same-direction comparison.
