True 3D / Piao-style route alignment summary

Stage: 20.65 true 3D / Piao-style geometry profile feasibility design

Scope and reading depth:
- This stage is feasibility design only. It did not run COMSOL, generate data, train a surrogate or inverse model, run refinement, update CURRENT_BASELINE.md, or create COMSOL baseline docs.
- Piao 2019 reading basis: this stage uses the existing project fullpaper alignment summary and the uploaded Piao PDF title/abstract/section-level context. It does not claim that the PDF full text was newly re-extracted in this run.
- Priewald 2013 reading basis: this stage uses existing project preflight/method-alignment summaries. It does not claim a new full-paper extraction in this run.
- External MFL method report basis: this stage uses existing project summaries and the available external report context. It does not claim a full new deepresearch pass.

Why the route changes now:
- Stage 20.61 expanded profile perturbation data showed single-height Bz COMSOL oracle residual ordering close to random.
- Stage 20.62 multi-height Bz did not improve ordering.
- Stage 20.63 same-direction Bx/By/Bz did not improve ordering.
- Stage 20.64 real multi-direction excitation gave a marginal Bz-only signal but failed the all-axis normalized gate and did not justify surrogate training.
- The current bottleneck is therefore not a small surrogate/loss/refinement setting. It is the identifiability of a 2D top-view profile under the current observation configuration.

Piao 2019 original route:
- Input: three-axis MFL signals.
- Feature stage: NLS-style physics feature extraction from characteristic MFL curves.
- Regressor: LS-SVM mapping from physics-based features to geometry/profile parameters.
- Output: 3D defect profile reconstructed from RBC six geometry parameters.
- RBC parameterization: L, W, D, wLD, wWD, wLW.

Migratable to this project:
- COMSOL-generated 3D profile labels, if variable-depth defect solids can be constructed.
- Bx/By/Bz observation schema already proven exportable in same-direction and multi-direction settings.
- Geometry parameter regression and profile reconstruction metrics.
- Projected-mask comparison with existing 2D mask metrics for continuity.
- Forward consistency as a later validation/refinement mechanism after oracle identifiability is shown.

Not directly migratable:
- Current Bz-only observations.
- Current 2D top-view mask/profile labels.
- Current rect/rot/polygon constant-depth geometry labels.
- Current profile perturbation residual objective.
- Any claim that KernelRidge/SVR weak adaptation is a full LS-SVM/NLS/RBC reproduction.
- Any claim that this stage fully reproduces Piao 2019.

Priewald 2013 alignment:
- The useful principle is forward-model-based inversion: candidate profile -> forward MFL -> observed MFL residual -> update/selection.
- Later Priewald-style refinement can use profile DOF regularization, forward residual, and constraints.
- Full nonlinear FEM Jacobian, Gauss-Newton/LM implementation, and analytic sensitivity are not prerequisites for 20.65 or 20.66.
- Refinement should not resume until true 3D oracle residual ordering is demonstrated.

Route decision:
- Stop 2D profile-forward small tuning as the main route.
- Set true 3D / Piao-style geometry profile feasibility as the next research route.
- Keep the dense mask baseline only as a comparator/reference, not the main route.
- The next executable stage should be a 20.66 smoke focused on RBC params -> depth map -> COMSOL variable-depth defect solid -> same-source projected mask -> Bx/By/Bz at 0.008 m -> delta check.
