# 第 7.15 步：v4 small polygon area-aware loss 扫描总结

## 实验目的

在第 7.14 步确认 `lambda_dice=0.03` 模型存在系统性面积高估后，加入可选 area-aware loss，尝试抑制 `pred_area > true_area` 和 polygon area_error 偏大的问题。

## 固定配置

* dataset = v4_balanced_complex
* loss_type = weighted_mse_dice_area
* defect_weight = 5
* lambda_dice = 0.03
* lambda_tv = 0
* epochs = 100
* physics_loss：未启用
* L-BFGS：未启用
* oversampling：未启用
* 模型结构：未修改
* evaluate_pinn.py 指标定义：未修改

## area-aware loss 定义

训练阶段使用 soft mask 计算面积：

```text
pred_soft_mask = clamp(1 - pred / defect_threshold, 0, 1)
target_mask = target < defect_threshold
pred_soft_area = sum(pred_soft_mask)
true_area = sum(target_mask)
area_loss = mean(abs(pred_soft_area - true_area) / (true_area + eps))
```

其中 `defect_threshold = MASK_THRESHOLD / MU_SCALE = 500 / 1000 = 0.5`，含义仍对应真实 `mu_r < 500`。

总损失：

```text
total_loss = weighted_mse_loss + lambda_dice * soft_dice_loss + lambda_area * area_loss + lambda_tv * tv_loss
```

本轮 `lambda_tv=0`。

## 与第 7.13B 无 area loss 候选对比

无 area loss 候选：`checkpoints/best_model_v4_smallpoly_w5_dice_0p03.pt`

* test IoU = 0.352032
* test Dice = 0.497474
* test area_error = 1.002027
* test area_ratio = 1.989118
* pred_area > true_area = 190 / 200
* polygon area_error = 1.478094
* small polygon pred_area=0 = 0 / 25
* small polygon IoU = 0.109439
* small polygon Dice = 0.183385
* medium polygon area_error = 1.891204
* multi_defect center_error = 0.877951

本轮推荐候选：`lambda_area = 0.05`

* test IoU = 0.350964，轻微下降；
* test Dice = 0.494989，轻微下降；
* test area_error = 0.794285，明显下降；
* test area_ratio = 1.774067，面积高估减轻；
* pred_area > true_area = 186 / 200，仍存在系统性高估但略有改善；
* polygon area_error = 1.197988，明显下降；
* small polygon pred_area=0 = 0 / 25，仍保持全部检出；
* small polygon IoU / Dice 高于无 area loss 候选；
* medium polygon area_error 从 1.891204 降到 1.429367；
* multi_defect center_error 从 0.877951 升到 0.967055，略有变差，但仍优于第 7.12B weighted MSE w5 的 1.087309。

## 结论

`lambda_area=0.05` 是本轮最合适的 area-aware loss 候选。它明显降低 overall area_error、polygon area_error 和 medium polygon area_error，同时 small polygon 仍保持 0/25 漏检。代价是 overall IoU / Dice 相比无 area loss 候选略有下降，multi_defect center_error 也略有变差。

area-aware loss 能改善 pred_area 系统性偏大，但没有彻底消除：`pred_area > true_area` 仍有 186 / 200。因此它适合作为 v4 small polygon / polygon area_error 专项候选，不应直接切换为全项目 baseline。

当前全项目推荐 baseline 仍为：

`checkpoints/best_model_v3_complex_tv_sweep_2e-6.pt`

## 下一步建议

建议先让 Claude Code review 第 7.15 步实现和结果记录。若继续优化，优先考虑：

1. 围绕 `lambda_area=0.05` 做小范围复核，例如 `0.04 / 0.05 / 0.07`；
2. 或按第 7.14 建议先验证 `defect_weight=3 / 4` 是否能进一步降低面积高估；
3. 暂不建议直接进入 focal loss 或 oversampling。
