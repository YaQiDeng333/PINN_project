## 6. 现有 external generator 缺口

现有 generator 的限制：

1. 只生成 single-line：
   - 配置里只有 `sensor_line.y_m`；
   - 没有 `scan_line_y` list；
   - 输出 shape 是 `(201,)`，不是 `(n_lines, L)`。

2. 只支持 rectangular_notch：
   - 没有 rotated_rect angle；
   - 没有 polygon vertices；
   - 不处理 single-defect 多形状族。

3. 不生成 mask：
   - 没有 `mask_x`；
   - 没有 `mask_y`；
   - 没有 `(H, W)` 2D / quasi-2D mask；
   - 没有 mask 与 COMSOL geometry 坐标映射说明。

4. 不生成目标 pack schema：
   - 没有 per-sample manifest；
   - 没有 `signal_kind`;
   - 没有 `n_lines`;
   - 没有 train/smoke split；
   - 没有目标 NPZ。

5. 当前 PINN_project 环境不能直接运行该 generator：
   - `mph` 未安装；
   - COMSOL / Java 环境变量未配置。

## 7. 最小可行方案 A：在 PINN_project 内调用 COMSOL

仅当明确要让 PINN_project 直接调用 COMSOL 时，最小修复路径如下。

需要先修：

1. 在一个专用 COMSOL Python 环境中安装并验证 `mph`；
2. 配置 COMSOL / Java runtime；
3. 确认 `comsol_start` 或 `mph.Client` 在 120 秒内稳定返回；
4. 决定使用 MCP tool 还是 Python `mph` 脚本，不要混用生成入口。

需要新增或改造脚本：

- `scripts/generate_comsol_single_defect_multiline_smoke_v1.py`

需要参数：

- `max_samples <= 3`
- `scan_line_y_m = [-0.004, 0.0, 0.004]`
- `sensor_x_start_m`
- `sensor_x_stop_m`
- `sensor_points`
- `sensor_z_m`
- `mask_x_start_m`
- `mask_x_stop_m`
- `mask_y_start_m`
- `mask_y_stop_m`
- `mask_shape`
- `geometry_params`

生成 1 个 sample 的流程：

1. 创建 no-defect COMSOL model；
2. 在全部 `(sensor_x, scan_line_y, sensor_z)` 点评价 `mf.Bz`，得到 `Bz_reference`；
3. 创建同 geometry settings 下的 defect model；
4. 在同一坐标点评价 `mf.Bz`，得到 `Bz_defect`；
5. 计算 `delta_Bz = Bz_defect - Bz_reference`；
6. 用同一 `geometry_params` 在 Python 中 rasterize 2D mask；
7. 保存 per-sample manifest；
8. 如果字段完整，再写 smoke NPZ；
9. 强制 `max_samples <= 3`，避免变成批量仿真。

当前不建议走方案 A，除非先完成环境修复。
