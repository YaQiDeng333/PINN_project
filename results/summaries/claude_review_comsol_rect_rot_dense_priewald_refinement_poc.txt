## 建议修复

1. **Dense mask 与 extracted geometry 的 test IoU 几乎完全相同**（0.5664 vs 0.5652）。geometry extraction 没有引入额外退化，但也说明 dense mask 本身预测质量不足以支持 extraction 得到更好的 rotated bbox。PCA + 最大连通分量 + fallback 策略在低质量 dense mask 下产生了很多错误分类（type accuracy 仅 0.5455，接近随机）。建议在进入 refinement 之前先解决 extraction 的质量问题。

2. **13/66 test 样本（20%）被分类为 surrogate_mismatch**，即 forward residual 下降但 mask/geometry 反而变差。这意味着 forward surrogate 在这些样本上引导了错误的方向。建议对 surrogate 在这些样本上的局部精度做评估，或对 surrogate 预测加入不确定性估计。

3. **Angle MAE 完全无效**（test ~21.3°，refinement 后无改善 delta +0.2°）。PCA 提取的角度质量极差，且 refinement 无法纠正。如果 angle 不具有可优化性，应考虑将 angle 从 refinement 参数中移除或 freeze。

## 是否认为 dense/coarse initializer + Priewald-style refinement 有实质价值

**有限价值**。方法论正确且 pipeline 无泄漏，但当前结果不足以证明 refinement 路线比直接改进 dense initializer 或 geometry head 更有效：

- Dense initializer test IoU 0.57，低于 20.51 geometry-head 0.61 和 dense single-defect baseline 0.65
- Refinement 仅带来 +0.016 IoU 改善（0.565→0.581），且仍低于 20.51 基准
- 22/66 test 样本要么退化要么 forward 改善但不转化为 geometry 改善
- 最大的收益来自 forward residual 下降（NRMSE -0.123），但 forward agreement ≠ geometry accuracy

---

## 是否建议继续 refinement 路线

**不建议在当前基础上继续**。当前瓶颈不是 refinement 的优化能力，而是：

1. Dense mask 初始质量不够 → geometry extraction 质量差 → refinement 起点的参数偏差太大
2. Forward surrogate 在 ~20% 样本上不可靠（surrogate mismatch），导致优化方向错误
3. Angle 基本不可优化，去掉 angle 后 refinement 只剩 4 个参数，价值进一步降低

---

## 下一步更建议什么

**优先提升 dense/coarse initializer 的 mask 质量或换用更强的 initializer**，然后再评估 refinement。

具体方向：
- 如果 dense mask test IoU 能达到 ≥0.65（接近 baseline），再重新测试 extraction + refinement 的效果
- 或者直接使用 20.51 geometry-head 输出作为 refinement 起点（该 head 本身 IoU 0.614 就优于当前 dense initializer 0.566 + refinement 0.581）
- 如果 surrogate mismatch 仍占 20%，考虑用 ensemble 或 uncertainty-weighted loss 替代当前的 MSE-only forward loss

---

## 简短原因

Pipeline 设计正确、无泄漏，但起始点太弱导致 refinement 的边际收益（+0.016 IoU）远小于起始点本身的差距（与 baseline 差 0.085 IoU）。在解决 initializer 质量之前，refinement 的增量价值无法被公平评估。当前最优 refined IoU 0.581 甚至低于不使用 refinement 的 20.51 geometry-head 0.614。
