COMSOL multi-axis profile perturbation preflight summary

Stage: 20.63 multi-axis MFL profile perturbation oracle ordering feasibility
Generated by: Codex multi-agent preflight + main-controller synthesis

1. 是否值得执行 20.63
Yes. 20.61 showed single-height Bz oracle residual is near random for profile quality ordering, and 20.62 showed multi-height Bz does not improve ordering. Multi-axis Bx/By/Bz is the next minimal richer-observation feasibility check before any surrogate training or profile refinement.

2. 是否已有结果否定 multi-axis observation route
No. Existing negative results only cover Bz-only and multi-liftoff Bz. They do not test Bx/By/Bz vector MFL observations.

3. 是否需要 COMSOL
Yes. Bx and By are not present in the existing pilot_v9, 20.61, or 20.62 NPZ schemas. Every selected row, including true_reference, must be real COMSOL forward for Bx/By/Bz.

4. COMSOL 是否支持 Bx / By / Bz 三轴导出
Code-level support exists through src/tools/results.py::_evaluate_at_points, which accepts an expression list and existing scripts already export multiple expressions such as ["mf.Bz", "mf.normB"]. However mf.Bx and mf.By have not been exercised in prior profile generators, so Stage B must run an expression probe before batch generation.

5. 实际使用的 COMSOL field expressions 是什么
Planned expressions: ["mf.Bx", "mf.By", "mf.Bz"], with units ["T", "T", "T"].

6. 是否只做 oracle residual ordering
Yes. Stage 20.63 does not train a surrogate, inverse model, or dense baseline, and does not run profile refinement.

7. 是否不训练 surrogate
Yes. Surrogate training is explicitly out of scope until COMSOL oracle residual ordering passes the multi-axis feasibility gate.

8. true_reference 是否也需要真实三轴 forward
Yes. Existing true_reference rows only have Bz arrays. They cannot provide Bx/By, so they are not valid multi-axis references. Stage 20.63 solves true_reference geometry with real COMSOL and exports Bx/By/Bz.

9. 唯一推荐执行方案
Generate a 24-base / 192-row multi-axis profile perturbation plan from the 20.61 expanded plan; run a hard Bx/By/Bz expression probe in the COMSOL generator; if the probe passes, generate a real 3-axis forward pack for all rows; then audit Bx-only, By-only, Bz-only, pairwise-axis, raw all-axis, and train-std-normalized all-axis oracle residual ordering.

10. 必须避免的问题
- If simultaneous Bx/By/Bz export fails, stop and write a blocker summary.
- Do not fall back to Bz-only or multi-height.
- Do not merge three independent scalar exports unless simultaneous export is proven stable; this implementation uses one expression-list export.
- Do not train a surrogate or run refinement in Stage 20.63.
- Do not reuse Bz-only true_reference arrays as primary multi-axis observations.
- Do not modify baseline docs.
