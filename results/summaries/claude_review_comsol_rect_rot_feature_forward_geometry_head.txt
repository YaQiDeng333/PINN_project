## Review: 20.51 feature-assisted neural geometry head + lightweight forward consistency

无。

### 建议修复

1. **REF_2050 值异常** — `REF_2050_IOU=0.5877` / `REF_2050_DICE=0.7351` 低于 `REF_2048` (0.5908 / 0.7385)，但 24accdb 的 commit message 是 "Improve COMSOL rect-rot neural geometry head POC"。如果 20.50 确实是对照实验，建议 double check 这些常数值是否正确，避免对比基准失真。

2. **forward surrogate 的 test abs_peak_index_error_mean = 27.91 偏高**（201 个采样点中峰值偏移近 28 个点），这可能限制 forward consistency loss 的质量。如果后续继续使用 forward consistency，建议提升 surrogate 的峰值对齐能力（例如加 spectral loss 或 peak-sensitive loss）。

3. **best epoch = 4 过早**（200 epoch 训练中第 4 epoch 就收敛到最佳 val score），说明当前架构在此任务上 capacity 或 learning rate schedule 可能不够合理。并非错误，但建议考虑 warm-up 或降低 lr 让训练更稳定。

### 是否认为 feature-assisted + forward consistency 有实质进步？

**有边际进步，但无实质突破。** IoU 从 0.5908 → 0.6138 (+0.023) 和 Dice 从 0.7385 → 0.7577 (+0.019) 是正向的，但 type accuracy 反而下降（0.6364 → 0.6212），且仍远低于 dense baseline (IoU 0.6515)。forward consistency 作为辅助 loss 有微弱正向作用（forward NRMSE 从 surrogate 直接评估的 0.463 降到 inverse forward 的 0.451），但未改变 type classification 这一核心瓶颈。

### 是否建议继续 direct neural geometry head？

**不建议。** 模型在 type classification 上倒退（61.9%→62.1% 仍在随机水平附近），wrong_type 是主导失败模式 (24/66)，这表明当前架构的 capacity/bias 不足以可靠地区分 rect vs rotated。继续小修小补（加更多 features、调 loss weights）大概率无法突破这个天花板。

### 是否建议转 Priewald-style coarse-to-fine refinement？

**建议。** 当前结果支持这个方向：mask segmentation (IoU 0.614) 已经有了一定基础，但精确几何参数估计（特别是 type 和 angle）是瓶颈。Priewald-style 的两阶段策略——粗分割→精修参数——天然适合处理 "形状对但类型错" 的失败模式（如 failure audit 中 rect_v9_0027 被误判为 rotated_rect 但 IoU 仍有 0.428）。

### 简短原因

Feature 辅助带来了小幅 IoU/Dice 提升，但 type classification 卡在 ~62%（仅略好于随机），说明 delta_bz-derived features 在区分矩形/旋转矩形这个核心任务上没有提供足够的判别信息。forward consistency 作为正则化有效但边际。直接 head 路线的天花板已基本可见，转到两阶段 coarse-to-fine 更可能突破当前精度瓶颈。
