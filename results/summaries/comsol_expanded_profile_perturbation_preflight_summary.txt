COMSOL expanded profile perturbation preflight summary

Stage: 20.61 expanded profile perturbation forward pack + profile-compatible surrogate calibration
Mode: multi-agent read-only preflight. No code changes, no experiments, no commits were made by subagents.

1. 是否值得执行 20.61
Yes. All subagents support continuing with expanded profile-native perturbation data. The 20.60 failure is consistent with insufficient base coverage rather than a decisive rejection of profile-native forward calibration.

2. 是否已有结果否定 expanded profile perturbation route
No. Prior results reject lossy profile-to-rect surrogate bridging, current small-pack profile-forward refinement, and small loss/config tweaks. They do not reject expanding profile-native perturbation forward data.

3. 是否需要 COMSOL
Yes. Stage A plan generation does not require COMSOL, but Stage B requires real COMSOL forward solves for new non-true-reference profile polygon rows.

4. COMSOL 是否支持 profile polygon geometry
Yes. 20.60 already used a profile polygon generator based on WorkPlane -> Polygon(table) -> Extrude, reusing polygon helper logic from generate_mfl_polygon_pilot_v5.py. This is a top-view profile footprint with constant-depth extrusion, not variable-depth 3D profile.

5. 是否需要训练 surrogate
Yes, after expanded pack generation. 20.60 wrote no reusable checkpoint and its surrogate gate failed, so 20.61 needs fresh profile-compatible surrogate calibration on the expanded pack.

6. 是否允许直接执行 expanded profile perturbation pack
Yes, if Stage A reaches at least the minimum 24 base / 192 rows plan and geometry validity checks pass. The target is 36 base / 288 rows. Stretch 48 / 384 is optional only if runtime is acceptable.

7. 唯一推荐执行方案
Build a 36-base / 288-row expanded profile perturbation plan with base-level split preservation and rect/rot balance. Reuse exact 20.60 rows only when perturb_sample_id or stable base+variant+geometry hash matches. Generate a cache-aware COMSOL expanded forward pack that reuses original true_reference rows, reuses exact 20.60 generated rows when safe, and solves all new non-reference profile polygon rows. Train exactly two profile-compatible surrogate candidates, select on validation only, and perform residual-ordering audit. Do not run refinement.

11. Stop conditions
- Stage A plan below 24 base / 192 rows.
- Missing train/val/test split or missing rect/rot in any split.
- Variant coverage incomplete without documented reason.
- Profile polygon invalid, self-intersecting, empty mask, or out of domain.
- COMSOL generation cannot reach minimum balanced usable rows.
- Row accounting fails: total_rows != reused_original_rows + reused_from_20_60_rows + real_comsol_forward_rows.
- Oracle residual ordering remains weak after expansion.
- Surrogate ordering/mismatch does not improve enough to pass gate.
