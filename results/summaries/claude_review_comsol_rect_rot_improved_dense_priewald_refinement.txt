## 建议修复

1. **`val_refinement_score` 过度依赖 forward NRMSE reduction。** 从 config sweep CSV 可见，所有 8 个 config 在 val 上的 `delta_mask_iou` 和 `delta_mask_dice` 均为负值，但 `val_refinement_score` 仍为正（因 forward NRMSE reduction ~0.056 的贡献远大于 mask delta 的惩罚）。选出的 "最佳" config 本质是"mask 质量退步最小的那个"，而非"真正改善几何质量的那个"。建议：当所有 config 的 mask delta 均为负时，直接判定 refinement 无效，而非选一个 least-bad config。

2. **Extraction 脚本重复训练了 dense initializer，导致与 standalone dense initializer summary 的 epoch/metrics 不一致。** Dense initializer summary 报 best epoch=12, val score=1.341162；但 extraction summary 报 best epoch=21, val score=1.322269。两者 seed 相同，差异来自 DataLoader shuffle 的非确定性。虽不影响正确性，但让读者困惑。建议 extraction 脚本接受一个已训练的 bundle 而非重新训练。

3. **Forward surrogate 的峰位误差较大（test abs_peak_index_error_mean = 27.9 / 201）。** 这是 surrogate mismatch 的物理根源：surrogate 在空间上偏移了 ~14% 的传感器长度，导致 refinement 优化 forward residual 时推动几何参数朝错误方向移动。

## Strong dense initializer + proposal extraction + Priewald refinement 是否有实质价值？

**Dense initializer + proposal extraction：有实质价值。** Test mask IoU/Dice 从 20.53 的 0.5664/0.7179 提升到 0.6689/0.7994（+10.2/+8.1 点）；extracted geometry test IoU/Dice 从 0.5652/0.7169 提升到 0.6726/0.8017（+10.7/+8.5 点）。Extraction 成功将 dense probability map 转为几何参数，且未引入明显退化。

**Priewald refinement：目前无实质价值。** Refinement 使 test IoU/Dice 从 0.6726/0.8017 降至 0.6646/0.7958（-0.8/-0.6 点）。Forward NRMSE 虽下降（0.4632→0.4049），但 geometry mask metrics 退步，属于典型的 surrogate mismatch。

## 是否建议继续 refinement 路线？

**建议暂停 Priewald-style refinement，先改善 forward surrogate。** 当前的 forward surrogate test NRMSE ~0.46、peak index error ~28/201 对于精确定位缺陷边界不够好。Refinement 在优化 forward residual 时，由于 surrogate 的空间偏差，会朝错误方向调整几何参数。逻辑链条：如果 surrogate 在空间上偏移了 ~14%，那么让 surrogate 更接近 observed signal 并不等价于让 geometry 更接近 true geometry。

## 下一步更建议什么

**A. 改善 forward surrogate。** 具体方向：
- 增大模型容量或改用卷积/PDE-informed 架构来降低空间误差
- 或将 forward surrogate 扩展为输出 field map（而非 3 条线）以提供更强的空间约束
- 目标：test NRMSE < 0.25，peak index error < 10

备选：如果 forward surrogate 难以短期内显著改善，考虑 **C. 直接做 mask/profile basis refinement**（跳过 surrogate，直接在 pixel/probability space 优化），因为 dense mask 已有 IoU 0.67 的强起点。

## 简短原因

Forward NRMSE 降了但 mask 质量退步 = surrogate 在引导优化走向错误方向。修复 surrogate 的质量（尤其是空间精度）是让 refinement 产生正向收益的前提条件。
