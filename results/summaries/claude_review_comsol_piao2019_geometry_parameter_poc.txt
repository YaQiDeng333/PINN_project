## 最终输出

### 必须修复
无。代码无 bug，文档 scope 声明准确一致，数据流正确（geometry_params → labels，delta_bz → features → regression → 硬光栅化 → 评估）。

### 建议修复

1. **type classifier 准确率过低（test 0.40）是系统性失败源**。failure audit 显示 60/600 样本为 wrong_type，其中许多 rect/rotated 被误分类为 polygon（导致 IoU=0）。建议在 summary 中更突出地标注 type accuracy 为该 POC 的首要瓶颈，并考虑是否增加针对 rect-vs-rotated 的二阶段分类器或增大 classifier 正则化力度。

2. **rotated_rect 角度 MAE ~18° 偏高**。考虑到 angle_sin/angle_cos 直接回归（不经归一化逆变换），可能存在 sin/cos 不一致（预测的 (sin, cos) 范数未必为 1）。`_angle_deg_from_sincos` 使用了 `atan2` 可容忍非单位范数，但回归器未约束 sin²+cos²=1。建议在 summary 中标注这一已知限制。

3. **`_feature_sets` 的安全检查略显脆弱**（`train_comsol_piao2019_geometry_parameter_poc.py:76`）。当前逻辑 `len(generic) == len(all_z)` 判断"无 NLS 特征"隐式依赖 generic 列数 ≠ 总列数。如果未来新增非 NLS 的 z_ 列，此检查不会误触发（generic 也会变多），但语义不够直接。建议改为显式检查是否存在 `_nls_` 列。

### 可以接受

- 代码结构清晰、无冗余，四脚本 pipeline 解耦合理（labels → features → train → evaluate）。
- NLS-style 指数拟合特征带来 +0.017 IoU / +0.018 Dice 的正向增益（虽小但非负），证明 Bz-only 指数拟合思路有一定信号。
- Failure audit 分类合理（wrong_type / wrong_angle / wrong_center / wrong_size / rasterizer_or_parameter_mismatch / polygon_unsupported），便于归因。
- 所有 scope 声明诚实、一致，没有越界声称。

### 是否认为该 revised POC 忠实参考了 Piao 2019 的核心思想，但未冒充完整复现？
**是。** 该 POC 正确捕捉了 Piao 2019 的核心 pipeline 思想（physics-inspired 指数特征提取 → ML 回归 → 几何参数 → 重建），并将每步诚实地降级为工程可用的替代方案（Bz-only 替代 tri-axis、指数拟合替代 NLS、KernelRidge/SVR 替代 LS-SVM、2D 矩形替代 RBC 3D 轮廓）。所有文档均一致声明 "weak adaptation, not reproduction"。

### 是否建议继续 Piao-style geometry parameter route？
**谨慎建议暂停（E），但不完全关闭。** 核心理由：
- NLS-style 特征增益微弱（+0.017 IoU），Bz-only 信息瓶颈可能是根本性限制（单一 Bz 分量无法唯一确定 5-7 个几何参数）。
- type accuracy 0.40 意味着分类器是严重瓶颈，而分类错误直接导致 IoU=0。
- 改善 Bz-only NLS feature extraction（选项 A）的边际收益可能很低，因为已经是穷举式手工特征工程（231 raw features）。

### 下一步更建议
**B. 加 differentiable rasterizer + neural geometry head。**
理由：手工特征 + 经典 ML 在该数据规模（train=402）和 Bz-only 信息限制下已接近上限。可微分光栅化器 + 轻量 neural geometry head 可以直接从 delta_bz 字段端到端学习几何参数，绕过手工特征工程和独立分类器的级联误差。这与 summary 自身的建议一致。

若选 B，建议同时保留当前 POC 的 geometry_params→hard rasterization 评估管线作为 baseline 对照。
