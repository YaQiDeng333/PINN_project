# PINN 漏磁反演项目术语说明

## 第 20 阶段术语补充

* COMSOL forward data

  指外部 `COMSOL_Multiphysics_MCP` 工程生成的真实 forward simulation 数据。当前用于构建 multi-line `delta_Bz` 输入和 2D / quasi-2D defect mask 标签，服务于提高 MFL 边界反演可辨识性。它不是 v3_complex 合成数据，也不直接更新 `CURRENT_BASELINE`。

* multi-line delta_Bz

  多条 scan line 上的 `delta_Bz = Bz_defect - Bz_no_defect`。当前 COMSOL pilot 数据使用 3 条 scan line：`[-0.001, 0.0, 0.001] m`，典型 shape 为 `(N, 3, 201)`。

* pilot pack

  小规模、真实、可审计、schema 完整的数据包，用于验证 ingest、loader、normalization、training gate 和 preview 链路。pilot pack 不等同于正式训练集，不能直接外推成正式模型性能结论。

* pilot_v2

  第 20.12 / 20.13 的 120-sample `rectangular_notch` COMSOL multi-line 数据包，split 为 80 / 20 / 20。它验证了较完整的 rectangular_notch 数据链路，但 defect_type 仍单一。

* pilot_v3

  第 20.14 / 20.15 的 48-sample `rotated_rect` / angle variation COMSOL multi-line 数据包，split 为 32 / 8 / 8，angle 覆盖 `-30, -20, -10, 10, 20, 30`。它用于验证 rotated geometry、mask angle consistency 和 rotated_rect training gate。

## 第 18.4 术语更新

* forward consistency

  在 mask-only grid decoder 训练中加入 frozen mask-to-Bz forward surrogate，使预测出来的 mask 不只要接近 true mask，还要能解释观测到的 Bz 信号。第 18.4 后，`lambda_forward=0.10` 的 forward consistency 已成为新的 `CURRENT_BASELINE`。

* mask-to-Bz forward surrogate

  独立训练的轻量前向替代模型，输入 defect mask / mask probability，输出对应 Bz signal。第 18.2 的 surrogate checkpoint 为 `checkpoints/mask_to_bz_forward_surrogate/best_mask_to_bz_forward_surrogate.pt`，在 forward consistency candidate 中冻结使用。

* Bz residual / Bz MSE

  用 frozen forward surrogate 从预测 mask 重建 Bz，再与观测 Bz 比较得到的误差。它用于判断预测 mask 是否能解释真实漏磁信号。第 18.4 baseline 的 Bz MSE 为 `0.1649 +/- 0.0135`，优于上一版 grid decoder reference 的 `0.3323 +/- 0.0149`。

* boundary-oriented CURRENT_BASELINE

  当前 boundary-oriented `CURRENT_BASELINE` 是 v3_complex mask-only grid decoder + forward consistency，`lambda_forward=0.10`，validation-selected probability threshold=`0.80`。它优先依据 IoU、Dice、area_error、`pred_area=0`、small / low-signal 以及 Bz consistency 选择，而不是 full μ-field MSE / MAE。

* 已知限制

  forward consistency 改善整体 shape metrics 和 Bz consistency，但 polygon area_error 相比上一版 grid baseline 轻微恶化，polygon / rotated_rect 精细边界圆斑化仍未根本解决。

## Step 15.4 terminology update

* boundary-oriented CURRENT_BASELINE

  After Step 15.4, this means the v3_complex mask-only grid decoder boundary model with validation-selected probability threshold `0.90`. It is selected for defect boundary inversion metrics, not for full-field MSE / MAE.

* mask-only grid decoder boundary model

  A mask-first model family that uses `BzEncoder`, projects the latent vector into a low-resolution 2D feature map, and upsamples with ConvTranspose2d / Conv2d blocks to output full-grid mask logits. It does not predict the full mu field.

* previous mask-only MLP boundary baseline

  The Step 15.1 / 15.2 coordinate-MLP mask-only boundary model. It is no longer the active CURRENT_BASELINE after Step 15.4, but is retained as a boundary reference.

* mu-threshold shape-oriented reference

  The composite-selection baseline retained for comparison when a full mu-field model is thresholded with raw `mu < 500`.

* MSE-oriented reference

  The `v3_complex_tv_sweep_2e-6` reference retained for full-field numeric error comparisons.

* known fine-boundary limitation

  The Step 15.4 grid decoder improves aggregate boundary metrics, but polygon and rotated_rect predictions can still look rounded or blob-like. It should not be interpreted as fully solving fine boundary reconstruction.

## 1. 项目基础术语

* PINN

  PINN 通常指 Physics-Informed Neural Network，物理信息神经网络。它不是只靠数据标签训练，还会尝试把物理规律放进训练目标里。本项目目前主线是从 Bz 漏磁信号反演二维 μ map；第六步已经加入过一个简化版物理一致性 Loss，但当前默认最佳模型仍是不启用物理 Loss 的 TV baseline。

* MFL / 漏磁检测

  MFL 是 Magnetic Flux Leakage，中文常说漏磁检测。它的基本思想是：材料里有缺陷时，磁场会在缺陷附近发生扰动，传感器测到的漏磁信号可以反映缺陷位置和形状。本项目用一维 Bz signal 来反演二维 μ map。

* Bz

  Bz 是磁场在 z 方向上的分量。在本项目中，`signals` 保存的就是一维 Bz 漏磁信号。模型输入不是直接输入完整二维图，而是输入 Bz signal 加空间坐标 `(x, y)`。

* μ map

  μ map 是二维磁导率分布图。可以理解为一张图，每个像素对应一个空间位置的磁导率。当前项目中背景区域约为 1000，缺陷区域约为 1。

* μ_r

  μ_r 是相对磁导率。代码和图里有时会写成 `mu` 或 `mu_r`。本项目的 `mu_maps` 实际保存的是二维相对磁导率标签。

* defect mask / 缺陷掩膜

  defect mask 是把 μ map 转成“缺陷区域 / 非缺陷区域”的二值图。本项目评估时通常使用：

  ```text
  mu < 500 视为缺陷区域
  mu >= 500 视为背景区域
  ```

  IoU、Dice、面积误差、中心误差这些指标都依赖 defect mask。

## 2. 数据集相关术语

* data_generator

  指数据生成脚本，当前主要是 `data_generator_v2.py`。它负责生成 Bz 信号、二维 μ map、缺陷类型和 metadata。

* train

  训练集。模型主要用 train 数据更新参数。当前简单缺陷数据为 `data/training_data_train.npz`，复杂缺陷第一版小样本为 `data/training_data_v3_complex_train.npz`。

* val

  验证集。用于训练过程中选择参数和保存最佳模型，例如按 val loss 或 val 指标决定哪个 checkpoint 更好。原则上主要用 val 集调参。

* test

  测试集。用于阶段性最终评估，不应该频繁用来调参。频繁看 test 集再调模型，会让 test 结果失去客观性。

* signals

  npz 文件中的一维 Bz 信号数组。形状通常类似：

  ```text
  signals = [样本数, x方向采样点数]
  ```

  当前默认网格下，单个 signal 长度是 200。

* mu_maps

  npz 文件中的二维 μ map 标签。形状通常类似：

  ```text
  mu_maps = [样本数, y方向网格数, x方向网格数]
  ```

  当前默认是 `[N, 100, 200]`。

* metadata

  每个样本的参数记录，例如缺陷类型、中心位置、宽高、角度、面积、深度、提离高度、噪声水平等。第七步后还加入了多缺陷和多边形相关字段。

* metadata_keys

  metadata 每一列或每个字段的含义说明。保存它的目的是以后读 npz 文件时，不需要回头猜 metadata 字段顺序。

* defect_types

  每个样本的缺陷类型，例如 `rect`、`circle`、`ellipse`、`triangle`，以及第七步新增的 `rotated_rect`、`polygon`、`multi_defect`。

* .npz

  NumPy 的压缩数据文件格式，可以在一个文件里保存多个数组。本项目的数据集都用 `.npz` 保存，例如：

  ```text
  data/training_data_train.npz
  data/training_data_v3_complex_train.npz
  ```

## 3. 模型结构相关术语

* train_pinn.py

  训练脚本。当前负责加载 train / val 数据，构建 BzEncoder + Fourier feature + MLP 模型，支持 TV Loss、物理一致性 Loss 初版和 L-BFGS refine。

* evaluate_pinn.py

  评估脚本。负责加载 test 数据和 checkpoint，输出 MSE、MAE、IoU、Dice、area_error、center_error，并保存对比图。

* BzEncoder

  Bz 信号编码器。它把一维 Bz signal 压缩成一个 latent vector，让后续 MLP 能结合这个全局信号信息来预测每个坐标点的 μ。

* latent vector

  隐向量。可以理解为模型从 Bz signal 中提取的一组特征数字。它不直接对应某个物理量，但包含模型认为有用的信息。

* Fourier feature

  坐标特征映射方法。直接输入 `(x, y)` 有时不容易学习复杂空间变化，所以会把坐标映射成多组 `sin` 和 `cos` 特征，帮助 MLP 表达更细的空间结构。

* MLP

  多层感知机，也就是由多层全连接层组成的神经网络。本项目中 MLP 接收 `[bz_latent, coord_features]`，输出每个坐标点的 μ 值。

* batch

  一次训练同时处理的一小组样本。比如 batch size = 4 表示每一步训练同时看 4 个样本。

* epoch

  一轮训练。一个 epoch 表示模型大致看完一遍训练集。训练日志中的 `Epoch 001/020` 表示总共训练 20 轮，目前是第 1 轮。

## 4. Loss 相关术语

* Loss

  损失函数，衡量模型预测和目标之间差多少。训练的目标就是让 loss 尽量变小。

* Loss 曲线

  把训练过程中每一轮的 loss 画成图。用于观察训练是否下降、是否震荡、是否过拟合。

* MSE Loss

  Mean Squared Error，均方误差。它会对预测 μ map 和真实 μ map 的差值平方后求平均。差得越大，惩罚越重。

* TV Loss

  Total Variation Loss，总变分损失。它鼓励 μ map 更平滑，减少毛刺、孤立噪点和背景碎斑。

* Total Variation

  可以理解为图像相邻像素之间变化的总量。变化越剧烈，TV 越大。TV Loss 会让预测图不要到处剧烈跳变。

* lambda_tv

  TV Loss 的权重。总损失里通常写成：

  ```text
  total_loss = mse_loss + lambda_tv * tv_loss
  ```

  当前推荐 baseline 使用 `lambda_tv = 5e-6`。

* physics_loss

  物理一致性 Loss。第六步中，它尝试让预测 μ map 经过简化 forward model 后得到的 Bz_pred 和输入 Bz signal 匹配。目前这个模块已跑通，但不作为默认最佳模型。

* lambda_phy

  physics_loss 的权重。第六步初版使用过 `lambda_phy = 1e-4`。当前推荐 baseline 里默认 `lambda_phy = 0`，也就是不启用物理 Loss 模型作为最佳模型。

* total_loss

  总损失。第六步初版中形式是：

  ```text
  total_loss = mse_loss + lambda_tv * tv_loss + lambda_phy * physics_loss
  ```

## 5. 优化方法相关术语

* Adam

  常用神经网络优化器。它比较稳定，适合常规训练。本项目大多数训练流程以 Adam 为基础。

* L-BFGS

  一种优化方法，常用于后期精修。本项目第五步已经跑通 L-BFGS refine，但当前小子集设置下泛化指标变差，所以不作为默认推荐方案。

* refine

  精修。意思是在已有模型基础上继续做一小段优化，而不是从头训练。L-BFGS refine 就是用 L-BFGS 对已有 TV 模型做后期调整。

* parameter sweep

  参数扫描。对多个候选参数分别训练或评估，然后比较结果。例如第 5.5 步扫描了多个 `lambda_tv` 和 L-BFGS 参数组合。

## 6. 评价指标相关术语

* MSE

  Mean Squared Error，均方误差。反映预测 μ map 和真实 μ map 的整体平方误差。MSE 对大误差很敏感。

* MAE

  Mean Absolute Error，平均绝对误差。反映预测和真实之间的平均绝对差，比 MSE 对极端误差没那么敏感。

* IoU

  Intersection over Union，交并比。用于比较预测缺陷 mask 和真实缺陷 mask 的重合程度。越接近 1 越好。

* Dice

  Dice 系数，也是衡量两个 mask 重合程度的指标。越接近 1 越好。它和 IoU 都是判断缺陷区域形状是否对的重要指标。

* area_error

  缺陷面积误差。比较预测缺陷面积和真实缺陷面积差多少。越小越好。

* center_error

  缺陷中心误差。比较预测缺陷中心和真实缺陷中心距离。越小越好。

* mask 类指标

  指依赖缺陷 mask 的指标，包括 IoU、Dice、area_error、center_error。它们更关心缺陷区域的形状、面积和位置，而不是每个像素的 μ 数值误差。

## 10. 当前项目中的重要判断原则

* MSE 下降但 IoU / Dice 变差

  说明模型整体像素误差变小了，但缺陷区域的形状重合变差了。对本项目来说，缺陷形状反演很重要，所以不能只看 MSE。第六步物理 Loss 初版就是这种情况：MSE 略好，但 IoU 和 Dice 变差，因此没有成为默认最佳模型。

* MAE 有改善但 mask 类指标没改善

  说明平均像素误差变小了，但缺陷区域的位置、面积或形状没有更准。第四步 TV Loss 初版曾出现类似情况，因此后续才做了 `lambda_tv` 参数扫描。

* L-BFGS 泛化指标变差

  说明 L-BFGS refine 可能把小子集拟合得更好，但在 val/test 上效果不稳定。当前项目中 L-BFGS 流程已跑通，但不作为默认推荐方案。

* test 集不要频繁调参

  test 集应该像最终考试一样使用。如果每次调参都看 test 结果，就相当于提前看答案，会导致最终指标不客观。本项目主要用 val 集选择参数，test 集用于阶段性最终评估。

* baseline 不应该随便覆盖

  baseline 是项目当前可信的对照点。即使某个新模型某个指标变好，也要看整体指标，特别是 mask 类指标。第 15.1 / 15.2 后，baseline 更新为 mask-only boundary model + threshold=0.90，是因为它在 IoU / Dice / area_error / `pred_area=0` / small / low-signal 上均优于 composite-selection reference。

* 新数据集不应该覆盖旧数据集

  旧数据集对应已有实验和模型指标。如果覆盖旧数据，之前的结果就很难复现。第七步复杂缺陷数据使用 `training_data_v3_complex_*.npz`，就是为了和旧的 `training_data_*.npz` 区分。

## 11. v3 / v4 复杂缺陷数据集相关术语

* v3_complex

  第三代复杂缺陷数据集。在原有 `rect`、`circle`、`ellipse`、`triangle` 基础上新增 `rotated_rect`、`polygon`、`multi_defect` 三类复杂缺陷。对应的数据文件为 `training_data_v3_complex_*.npz`。当前 boundary-oriented baseline 为 mask-only boundary model + threshold=0.90；`checkpoints/best_model_v3_complex_tv_sweep_2e-6.pt` 仅保留为 MSE-oriented reference。

* v4_balanced_complex

  第四代平衡复杂缺陷数据集。在 v3_complex 基础上做了数据增强和样本平衡：提高 5 顶点 polygon 比例、平衡 2/3 缺陷 multi_defect（约 40%/60%）、平衡 complexity_level（约 30%/40%/30%）、过滤极弱信号 polygon。对应的数据文件为 `training_data_v4_balanced_complex_*.npz`。

* rotated_rect

  旋转矩形缺陷。和 `rect` 不同的是它有旋转角度，边界不是轴对称的。在 v3/v4 数据集中是三类复杂缺陷之一。

* polygon

  多边形缺陷。由 4 到 9 个顶点围成的任意凸多边形，用于模拟不规则形状的真实缺陷。在 v3/v4 数据集中是当前模型表现最差的缺陷类型，尤其 small polygon 容易漏检。

* multi_defect

  多缺陷样本。一个样本内包含 2 或 3 个独立缺陷（从 circle、ellipse、rect、triangle 中随机选取组合）。当前模型对 3 缺陷 multi_defect 的 center_error 明显高于 2 缺陷。

* complexity_level

  样本复杂度级别。v3/v4 数据集中分为 1/2/3 级：
  - level 1：简单缺陷（circle、ellipse、rect、triangle）
  - level 2：中等复杂度（rotated_rect、polygon）
  - level 3：高复杂度（multi_defect）

  复杂度越高，模型的 MSE、MAE、area_error、center_error 通常越差。

* num_defects

  多缺陷样本中的缺陷个数。v4 数据集中为 2 或 3。模型在 3 缺陷样本上的 center_error 明显大于 2 缺陷。

* num_vertices

  polygon 缺陷的顶点数。v3/v4 数据集中从 4 到 9。诊断发现并非顶点越多越差：5 顶点 polygon 的 IoU / Dice 反而最低。

* component_types / component_centers / component_sizes / component_angles

  multi_defect 样本中每个子缺陷的类型、中心坐标、尺寸和角度。这些字段保存在 metadata 中，用于分析多缺陷样本中不同组件对误差的影响。

* polygon_vertices

  polygon 缺陷的顶点坐标列表，保存在 metadata 中。用于生成和检查 polygon 形状。

* min_mu

  样本中缺陷区域的 μ 最小值。复杂缺陷区域 μ 值不固定（旧简单缺陷默认约为 1），v3/v4 中引入了深度变化，`min_mu` 记录了该样本最小 μ 值。

* mask_pixels

  polyon 缺陷 mask 中的像素数量，即缺陷区域的实际像素面积。v4 数据集中用于将 polygon 按面积分为 small / medium / large 三个 bin。同时作为 polygon 质量控制条件：`mask_pixels >= 30`。

* signal_peak_to_peak

  Bz 信号的峰峰值，即信号最大值减最小值。v4 数据集中用于辅助判断 polygon 信号是否足够强。

* signal_snr

  信号信噪比。v4 数据集中用于筛选 polygon：`signal_snr >= 5` 才会保留。低于此阈值的 polygon 信号太弱，模型几乎不可能检出。

* area_bin

  polygon 按缺陷像素面积划分的档位。v4 中使用：
  - `small`：`mask_pixels < 120`
  - `medium`：`120 <= mask_pixels < 500`
  - `large`：`mask_pixels >= 500`

  small polygon 是当前模型最难的子集，在 TV-only 训练中 IoU 和 Dice 常为 0。

* balance_group

  v4 数据集中每个样本所属的平衡分组标签。用于辅助检查各类缺陷的分布是否接近目标比例，不直接参与训练。

* seed

  随机种子。v4 数据集使用 `seed=7904` 生成。固定 seed 可以保证数据集的复现性。

* --dataset

  `train_pinn.py` 和 `evaluate_pinn.py` 的命令行参数。用于选择训练或评估使用的数据集，可选值包括 `simple`、`v3_complex`、`v4_balanced_complex`。不同的 dataset 会自动读取对应的 train / val / test npz 文件。

* --complex

  `data_generator_v2.py` 的命令行参数。用于生成 v3_complex 复杂缺陷数据集。已不建议直接使用，推荐用 `--dataset v4_balanced_complex`。

* --mode

  `train_pinn.py` 的命令行参数。控制训练模式，可选值包括 `adam_tv`（Adam + TV Loss）、`adam_tv_phy`（Adam + TV Loss + 物理 Loss）。默认值为 `adam_tv`。

* --loss-type

  `train_pinn.py` 的命令行参数。选择训练用的 loss 类型，可选值包括：
  - `mse`：纯 MSE Loss（默认值，保持不变）
  - `weighted_mse`：缺陷像素加权 MSE
  - `weighted_mse_dice`：加权 MSE + soft Dice Loss
  - `weighted_mse_dice_area`：加权 MSE + soft Dice + area-aware Loss

* MASK_THRESHOLD / MU_SCALE

  代码中的常量。`MASK_THRESHOLD` 是缺陷 mask 的 μ 阈值，当前值为 500（对应 `mu_r < 500` 视为缺陷）。`MU_SCALE` 是 μ 值的归一化缩放因子。训练代码中阈值使用 `MASK_THRESHOLD / MU_SCALE` 得到归一化后的阈值。

* –epochs

  `train_pinn.py` 的命令行参数。控制训练总轮数。不同实验使用过 20、50、100 epoch。

## 12. 进阶 Loss 相关术语

* weighted_mse / defect_weight

  缺陷像素加权 MSE Loss。在计算 MSE 时给缺陷区域像素更高的权重，让模型更关注缺陷区域而不是背景。`defect_weight` 是缺陷像素相对背景像素的权重倍数。当前 v4 small polygon 专项实验中 `defect_weight=5` 是最平衡的候选。

  启用方式：`--loss-type weighted_mse --defect-weight 5`。

* soft Dice Loss

  一种基于 Dice 系数的连续可导 loss，用于直接优化缺陷区域的重合度。和评价用的 hard Dice（基于二值 mask）不同，soft Dice 作用于模型输出的连续值，因此可以作为训练 loss。当前实验中 `lambda_dice=0.03` 是 v4 small polygon 专项候选推荐值。

  启用方式：`--loss-type weighted_mse_dice --lambda-dice 0.03`。

* lambda_dice

  soft Dice Loss 的权重系数。总 loss 中形式为 `total = weighted_mse + lambda_dice * soft_dice_loss`。值越大，模型越倾向于提升 Dice 重合度，但可能导致面积过分割。

* area-aware Loss

  面积感知 Loss。基于预测 μ map 的 soft mask 面积与真实面积的相对误差设计，用于惩罚预测面积过大或过小。当前主要用于解决 `lambda_dice=0.03` 模型中系统性面积高估的问题。

  启用方式：`--loss-type weighted_mse_dice_area --lambda-area 0.04`。

* lambda_area

  area-aware loss 的权重系数。总 loss 中形式为 `total = weighted_mse + lambda_dice * soft_dice + lambda_area * area_loss`。值越大，面积约束越强。

* symmetric / over_only（面积 Loss 类型）

  area-aware Loss 的两种计算模式：
  - `symmetric`（默认）：预测面积偏大和偏小都惩罚，倾向让面积尽量匹配。
  - `over_only`：只惩罚预测面积偏大（`relu(pred_area - true_area)`），不关心偏小。`over_only` 可以更有效减少面积高估，但会导致 small polygon 漏检回升，因此当前不作为推荐方案。

  启用方式：`--area-loss-type over_only`。

* --lambda-tv

  `train_pinn.py` 中 TV Loss 权重的命令行参数。推荐值为 `5e-6`（simple 数据集）或 `2e-6`（v3_complex 数据集）。当前 v4 实验中因使用 `defect_weight` 和 soft Dice 等替代策略，TV Loss 权重通常设为 0。

* --lambda-phy

  `train_pinn.py` 中物理一致性 Loss 权重的命令行参数。目前仅在第六步实验中使用过 `1e-4`，不作为默认推荐。

## 13. 训练策略和实验概念

* 漏检 / pred_area=0

  漏检指模型完全没有预测出缺陷区域，即预测的缺陷 mask 面积为零（`pred_area = 0`）。这是当前 v4 small polygon 面临的最大问题之一：TV-only 训练下 small polygon 几乎全部漏检。后续通过 defect-weighted MSE 和 soft Dice Loss 逐步解决。

  在 v4 balanced_complex test 集的 25 个 small polygon 样本中：
  - TV-only 训练：全部漏检（`pred_area=0`：25/25）
  - `defect_weight=5`：12/25 漏检
  - `defect_weight=10`：0/25 漏检
  - `lambda_dice=0.03`：0/25 漏检

* 过分割 / pred_area > true_area

  过分割指模型预测的缺陷面积系统性地大于真实缺陷面积。当前 `lambda_dice=0.03` 模型中 200 个 test 样本有 190 个 `pred_area > true_area`，mean area_ratio ≈ 1.99。area-aware loss 和 `over_only` 类型都是为了缓解这个问题而引入的。

* area_ratio / area_bias

  面积比，即 `pred_area / true_area`。area_ratio = 1 表示预测面积恰好匹配真实面积；> 1 表示过分割；< 1 表示欠分割。在 area_error 诊断中，mean area_ratio = 1.99 说明模型平均预测面积几乎是真实的 2 倍。

* 延长训练 / long training

  指将 epoch 从常规的 20 或 50 增加到 100 或更多。第 7.7 步验证了 v3_complex 100 epoch 长训练：MSE 和 center_error 略有改善，但 MAE、IoU、Dice、area_error 变差，因此长训练不作为默认推荐。

* oversampling

  过采样策略，指在训练时对某些难样本（如 small polygon）增加采样频率。在多项下一步建议中被提到，但截至第 7.17 步仍处于讨论阶段，尚未实际应用。

* focal loss

  一种处理类别不平衡的 Loss 函数，对容易分类的样本降低权重，对难样本提高权重。在多项下一步建议中被提到，但截至第 7.17 步尚未实际应用。

* forward model

  物理前向模型。在 physics_loss 中，它是从 μ map 计算 Bz 信号的简化物理公式。当前项目中使用的是简化版，不是完整的三维电磁模拟。第六步 physics_loss 初版已跑通但不优于 baseline。

* Bz_pred / Bz reconstruction error

  `Bz_pred` 是物理前向模型从预测 μ map 计算出的 Bz 信号。`Bz reconstruction error` 是 `Bz_pred` 和输入 `Bz signal` 之间的误差，在 physics_loss 中直接用作物理一致性惩罚项。第六步初版中该误差约为 0.0815。

* per-sample evaluation

  逐样本评估。对测试集中每个样本单独记录 MSE、IoU、Dice 等指标，保存到 CSV 文件中。用于按缺陷类型分组统计和找出表现最差的单样本。输出文件例如 `evaluation_metrics_*.csv`。

* seed / repeat

  随机种子和重复实验。当前项目训练流程尚未提供 `--seed` 参数，因此相同配置不同次训练可能得到略有不同的结果。NEXT_STEP 中多次提到如果继续做 loss 对比，需要加入 seed / repeat 验证来消除训练随机性的影响。

* .pt

  PyTorch 模型文件格式。本项目所有模型权重都保存为 `.pt` 文件，位于 `checkpoints/` 目录下。

## 15. CURRENT_BASELINE failure-driven audit 相关术语

* CURRENT_BASELINE

  当前项目认可的默认对比基线。它通常包括模型 checkpoint、数据集版本、关键超参数和 test 指标。后续实验即使在某些局部指标上更好，也不能自动替换 CURRENT_BASELINE，必须满足主线定义的接受条件。第 15.1 / 15.2 后，CURRENT_BASELINE 当前按项目目标定义为 boundary-oriented best baseline。

* boundary-oriented baseline

  面向缺陷边界形状反演目标的 baseline。它优先关注 IoU、Dice、area_error、`pred_area=0`、small 和 low-signal 表现。当前 v3_complex boundary-oriented CURRENT_BASELINE 是 mask-only boundary model，并使用 validation-selected probability threshold=0.90。

* shape-oriented baseline

  面向形状反演目标的 baseline。它优先关注 IoU、Dice、area_error、`pred_area=0` 等缺陷 mask / boundary 指标，而不是只追求 MSE / MAE 最低。第 13.4 / 13.5 的 composite-selection candidate 当前保留为 μ-threshold shape-oriented reference。

* mask-only boundary model

  独立的边界候选模型，直接从 Bz 输入预测 defect probability / mask，不再预测完整 μ field，也不通过 `pred_mu < 500` 间接得到 mask。因此 MSE / MAE 不适合作为它的主指标。

* MSE-oriented reference baseline

  面向数值误差的参考 baseline。它在 MSE / MAE 上更好，但不一定最符合缺陷边界形状反演目标。旧模型 `checkpoints/best_model_v3_complex_tv_sweep_2e-6.pt` 当前保留为 MSE-oriented reference baseline。

* selection_metric

  训练时用于选择 best checkpoint 的验证集指标。默认 `selection_metric=mse` 保持旧逻辑；第 13.4 / 13.5 使用 `selection_metric=composite`，其中 `composite = val IoU + val Dice - val area_error`，更偏向选择 shape / mask 指标更好的 checkpoint。

* gate experiment

  快速门控实验。它不是完整正式实验，通常只跑一个 seed 或一个小配置，用来判断某个方向是否值得扩大。如果 gate 失败，就停止该方向，不继续做 3 seed、参数扫描或更复杂实现。

* failure audit

  失败样本审计。对 CURRENT_BASELINE 的 test 结果逐样本分析，找出 IoU、area_error、center_error 最差的样本，以及 `pred_area=0`、面积高估或面积低估等主要失败模式。

* prediction distribution audit

  预测分布审计。分析模型输出的 `pred_mu` 在真实缺陷像素和背景像素上的分布，例如有多少缺陷像素低于 500、有多少落在 500-600、有多少高于 600。它帮助判断问题是阈值附近的校准偏差，还是模型完全没有识别缺陷。

* threshold calibration

  阈值校准。只在评估阶段改变 mask threshold，例如从 `pred_mu < 500` 改成 `pred_mu < 600`，或对 mask-only probability 使用 validation-selected threshold=0.90，观察 IoU、Dice、area_error 是否改善。它不改变模型权重，因此属于 evaluation-level calibration，不等同于训练改进。

* validation-selected probability threshold

  只用 validation set 选择的全局 mask probability 阈值。第 15.2 中三个 mask-only seed 共享同一个 threshold=0.90；test set 只用于最终评估，不参与阈值选择。

* mask threshold

  把连续的 μ 预测转成缺陷 mask 的阈值。本项目标准定义是 `mu_r < 500` 判定为缺陷；如果使用归一化 μ，则对应 `mu_norm < 0.5`。

* pred_area / true_area

  `pred_area` 是预测缺陷 mask 的面积，`true_area` 是真实缺陷 mask 的面积。二者比较可以判断模型是否漏检、低估或高估缺陷面积。

* pred_area=0

  预测缺陷面积为 0，表示模型完全没有预测出缺陷区域。它通常对应漏检，尤其会显著伤害 small defect 的 IoU 和 Dice。

* pred_area<true_area / pred_area>true_area

  `pred_area < true_area` 表示面积低估，模型预测的缺陷区域比真实缺陷小；`pred_area > true_area` 表示面积高估，模型预测区域过大。第 15.2 的 mask-only threshold=0.90 主要用于压低 fixed threshold=0.5 时的系统性面积高估。

* small / medium / large area bin

  按真实缺陷面积把样本分成小、中、大三个面积分桶。分桶通常基于当前数据集的 true_area 分位数，而不是固定物理尺寸。这样可以观察模型是否主要失败在 small defect 上。

* small-defect oversampling

  小缺陷过采样。训练时复制 small defect 样本，让模型更频繁看到小面积缺陷。第 10.8 的 v3_complex gate 表明，简单 oversampling 没有改善 small defect，反而使面积低估更严重。

* signal difficulty audit

  信号难度审计。分析输入 Bz signal 的强度特征，例如 `max_abs_bz`、`peak_to_peak_bz`、`mean_abs_bz`、`std_bz`、`l2_energy_bz`，并与 IoU、Dice、`pred_area=0` 对比。第 10.9 显示 small defect 和 `pred_area=0` 样本通常具有更弱的 Bz 信号，但信号强度不能完全解释所有失败。

* data observability audit

  数据可观测性审计。它不训练模型，而是直接分析数据本身：缺陷面积、缺陷中心、Bz 信号强度、Bz 峰值位置和缺陷中心之间的偏移。第 11.2 用它判断 v3_complex 的 small / low-signal 样本是否在单条 Bz 输入下天然更难辨识。

* low-signal sample

  低信号样本。通常指 Bz 信号强度处在数据集中较低分位的样本，例如第 11.2 中使用 train set 的 `max_abs_bz` 最低 1/3 作为 low-signal 定义。low-signal 不等于一定不可识别，但它往往更容易出现低 IoU 或 `pred_area=0`。

* Bz peak x

  Bz 输入信号中绝对值最大的峰值所在 x 坐标。它可以粗略反映漏磁响应最强的位置，但不一定等于缺陷几何中心，尤其在 multi_defect 或不规则缺陷中会发生偏移。

* defect centroid x

  真实缺陷 mask 的 x 方向几何中心。本项目通常从 `mu_map < 500` 的真实缺陷 mask 中计算。

* peak-centroid offset

  `abs(Bz peak x - defect centroid x)`，表示 Bz 最强响应位置和真实缺陷几何中心之间的距离。偏移越大，说明单条 Bz 信号的峰值位置越不能直接代表缺陷中心。

* raw_plus_norm_stats

  第 11.3 gate 中临时测试的 Bz 输入特征模式：保留原始全局归一化 Bz 输入，同时加入每个样本按自身最大绝对值归一化后的 Bz 形状通道，以及 `max_abs_bz`、`peak_to_peak_bz`、`l2_energy_bz` 等简单统计量。该 gate 失败后没有保留为主线代码分支。

* Bz input feature augmentation

  Bz 输入特征增强。指不改变 decoder 和 loss，只改变 Bz 输入到 encoder 之前的表示方式，例如加入样本内归一化形状或信号强度统计量。第 11.3 说明简单输入增强没有改善 v3_complex 的 small / low-signal 样本。

## 16. COMSOL MFL 数据接入术语

* COMSOL MCP

  指 `COMSOL_Multiphysics_MCP` 仓库提供的 Model Context Protocol 工具层，用于让 Codex 通过 MCP 调用 COMSOL 建模、求解和导出结果。当前 PINN_project 只接收它导出的轻量 CSV / JSON / TXT 结果，不直接启动 COMSOL，也不把 `.mph` 模型文件纳入训练数据目录。

* MFL forward model

  漏磁正问题模型。给定几何、材料、磁场源、网格和 study 后，用 COMSOL 计算 sensor line 上的 Bz / normB 响应。它和 PINN 反演方向相反：forward model 是从缺陷参数到磁场信号，PINN 主线目标是从 Bz 信号反演缺陷相关的 μ map。

* no_defect baseline

  无缺陷基线模型。它使用完整 steel block，没有 notch / void，用同一条 sensor line 导出 Bz / normB。后续有缺陷样本会和它相减，得到缺陷引起的相对扰动。

* rectangular_notch

  当前 COMSOL 小规模 sweep 使用的矩形缺陷类型。它表示 steel block 中的简单 rectangular notch / void，用 width、depth、length 和 center 位置描述。

* sensor line

  位于试件上方的一维采样线，用于导出磁场响应。当前接入数据的 sensor line 为 `x=-0.04..0.04 m, y=0, z=0.002 m`。

* fixed 201 points

  固定 201 个 sensor line 采样点。它避免直接依赖 COMSOL 网格导出导致的点数不一致，便于后续批量样本对齐和输入张量构造。

* Bz

  磁通密度在 z 方向上的分量。当前 COMSOL CSV 中同时保存 `Bz_no_defect` 和 `Bz_defect`，分别对应无缺陷 baseline 和有缺陷模型。

* normB

  磁通密度向量模长。当前 COMSOL CSV 中保存 `normB_no_defect` 和 `normB_defect`，用于观察总磁场强度的变化。

* delta_Bz

  `Bz_defect - Bz_no_defect`。它表示 rectangular notch 相对 no_defect baseline 引起的 Bz 扰动，是判断缺陷响应是否可观察的核心对比量之一。

* metadata.csv

  COMSOL scripted sweep 输出的样本索引表。当前包含 sample_id、defect_type、缺陷尺寸、缺陷中心、sensor_z、max_abs_delta_Bz、求解状态和输出文件路径等字段。复制到 PINN_project 后，`output_csv` 字段仍保留原始 COMSOL 输出路径，本地读取应以同目录下的 `sample_XXX.csv` 为准。

* scripted sweep

  脚本化参数扫描流程。当前对应 `COMSOL_Multiphysics_MCP/scripts/generate_mfl_rectangular_sweep.py`，它按配置生成 no_defect baseline 和 5 个 rectangular notch 样本，并导出固定 201 点 CSV、metadata 和 sweep summary。

* simplified ExternalCurrentDensity source

  当前 COMSOL smoke / small sweep 使用的简化磁场源。它能产生非零 MFL 响应，但不是实际 MFL 传感器或磁化装置，因此这批数据只能用于流程验证，不能直接视为实验级真实训练数据。

## 17. Geometry-Aware / Forward-Consistent 反演术语

* forward consistency

  前向一致性。要求预测出的 mask 或 geometry 通过 frozen forward surrogate 后，能够解释观测到的 Bz / `delta_bz`。在 v3_complex `CURRENT_BASELINE` 中，它是 mask -> Bz surrogate residual；在第 20.51 中，它是 predicted geometry -> lightweight forward surrogate -> `delta_bz` residual。test set 只能用于最终评估，不能参与 forward loss 权重或 threshold 选择。

* forward surrogate

  可微的轻量前向代理模型。它用已有数据学习从 mask 或 geometry 参数到 Bz / `delta_bz` 的映射，训练好后冻结，用于 inverse model 的 consistency loss 或 refinement objective。它不是 COMSOL 本体，也不是新的仿真数据生成器。

* geometry head

  从 Bz encoder latent 直接输出显式几何参数的网络头，例如 center、width、length、depth、angle 和 type probability。第 20.48-20.51 说明 rect/rot geometry labels 与 rasterizer 可用，但 direct geometry head 的 type / angle 学习仍不足，不应继续做小修补。

* differentiable rasterization

  可微光栅化。把连续几何参数转换成 soft mask 的过程，并允许梯度从 mask loss 反传到几何参数。当前 rect/rot POC 使用 PyTorch soft rotated-rectangle SDF rasterizer，固定 temperature `0.0005`，true geometry raster IoU 可达到 1.0000。

* Piao-style weak adaptation

  对 Piao 2019 的弱适配探索。原论文核心是三轴 MFL、Rational Bezier Curve 3D profile、NLS 物理特征和 LS-SVM。当前项目第 20.47-revised 只做 Bz-only / 2D-quasi-2D adaptation：hand-crafted + NLS-style features 加 SVR/KRR/Ridge，不是完整复现。该路线没有通过 acceptance。

* Priewald-style refinement

  受 Priewald 类 forward-model inversion 启发的低维几何精修路线。当前不计划复现完整 FEM Jacobian 或 Gauss-Newton 工程，而是先用 coarse mask / geometry proposal，再在低维 geometry 参数空间中用 frozen forward surrogate residual 精修。这是 20.51 后优先方向。

* component_count=2 / component_count=3

  COMSOL true multi_defect 数据中的组件数量。`component_count=2` 表示每个样本包含两个独立缺陷组件，`component_count=3` 表示三个独立组件。二者都要求 `connected_component_count` 与组件数一致，并且使用 true joint COMSOL response，不允许用 single-defect Bz 相加冒充。

* COMSOL_DATA_BASELINE_V3 candidate

  将 single-defect、`component_count=2` multi_defect、`component_count=3` multi_defect 合并为统一 COMSOL data-domain benchmark 的候选包。第 20.42 数据包和 schema 通过 review，但 lightweight combined decoder 因 cc2 明显退化而未通过 baseline acceptance，因此没有创建 `COMSOL_DATA_BASELINE_V3.md`。
# PINN 漏磁反演项目术语说明

## 第 20.65 路线术语补充

* true 3D / Piao-style route

  指第 20.65 之后的当前研究主线：不再继续 2D / quasi-2D profile-forward surrogate 和 refinement 小修，而是验证能否生成真实 variable-depth 3D defect profile，并用 Bx / By / Bz 响应反演 3D 几何参数。它是 feasibility route，不等于已经有 train-ready true 3D dataset。

* Piao RBC six parameters

  Piao-style 第一版候选标签。RBC = Rational Bezier Curve，用六个低维几何参数描述 3D 缺陷 profile：`L`、`W`、`D`、`wLD`、`wWD`、`wLW`。第一个 pilot 推荐把这六个参数作为主标签，depth grid / projected mask 只作为派生监督和 QA。

* variable-depth true 3D defect solid

  指缺陷深度随 `(x, y)` 位置变化的真实 3D defect volume，而不是把 top-view footprint 以单一 depth 向下 extrude 成 constant-depth prism。当前 COMSOL 支持真实 3D volume solve，但 RBC / variable-depth solid 生成尚未验证，是第 20.66 smoke 的核心 blocker。

* projected 2D mask from 3D profile

  从 3D depth map 投影得到的 2D footprint mask。第 20.65 固定的规则是：`RBC params + pose -> depth(x,y) -> footprint = depth > max(1e-6 m, 0.01 * D) -> 2D mask`。它用于兼容旧的 IoU / Dice / area_error 指标，不代表主任务仍是 2D mask。

* 20.66 smoke pack

  下一阶段建议的小规模几何链路验证，不是正式训练数据集。目标只验证 `RBC params -> depth map -> COMSOL variable-depth defect solid -> same-source projected mask -> Bx/By/Bz @ sensor_z_m=0.008 -> delta_B = B_defect - B_no_defect`。不要在 smoke 阶段引入 multi-height；`0.012m` 只作为后续 pilot 或 ablation 设计保留。

* preliminary acceptance guidance

  指第 20.67 pilot 可能使用的初步验收参考，例如 projected mask IoU `>=0.65`、Dice `>=0.78`、profile error `<=0.25`。这些阈值尚未被 true 3D 数据验证，不能写成硬标准。
