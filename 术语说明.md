# PINN 漏磁反演项目术语说明

本文档用于解释本项目中 README、实验日志和报告里反复出现的术语。解释尽量结合当前项目，不按论文写法展开。

## 1. 项目基础术语

* PINN

  PINN 通常指 Physics-Informed Neural Network，物理信息神经网络。它不是只靠数据标签训练，还会尝试把物理规律放进训练目标里。本项目目前主线是从 Bz 漏磁信号反演二维 μ map；第六步已经加入过一个简化版物理一致性 Loss，但当前默认最佳模型仍是不启用物理 Loss 的 TV baseline。

* MFL / 漏磁检测

  MFL 是 Magnetic Flux Leakage，中文常说漏磁检测。它的基本思想是：材料里有缺陷时，磁场会在缺陷附近发生扰动，传感器测到的漏磁信号可以反映缺陷位置和形状。本项目用一维 Bz signal 来反演二维 μ map。

* Bz

  Bz 是磁场在 z 方向上的分量。在本项目中，`signals` 保存的就是一维 Bz 漏磁信号。模型输入不是直接输入完整二维图，而是输入 Bz signal 加空间坐标 `(x, y)`。

* μ map

  μ map 是二维磁导率分布图。可以理解为一张图，每个像素对应一个空间位置的磁导率。当前项目中背景区域约为 1000，缺陷区域约为 1。

* μ_r

  μ_r 是相对磁导率。代码和图里有时会写成 `mu` 或 `mu_r`。本项目的 `mu_maps` 实际保存的是二维相对磁导率标签。

* defect mask / 缺陷掩膜

  defect mask 是把 μ map 转成“缺陷区域 / 非缺陷区域”的二值图。本项目评估时通常使用：

  ```text
  mu < 500 视为缺陷区域
  mu >= 500 视为背景区域
  ```

  IoU、Dice、面积误差、中心误差这些指标都依赖 defect mask。

## 2. 数据集相关术语

* data_generator

  指数据生成脚本，当前主要是 `data_generator_v2.py`。它负责生成 Bz 信号、二维 μ map、缺陷类型和 metadata。

* train

  训练集。模型主要用 train 数据更新参数。当前简单缺陷数据为 `data/training_data_train.npz`，复杂缺陷第一版小样本为 `data/training_data_v3_complex_train.npz`。

* val

  验证集。用于训练过程中选择参数和保存最佳模型，例如按 val loss 或 val 指标决定哪个 checkpoint 更好。原则上主要用 val 集调参。

* test

  测试集。用于阶段性最终评估，不应该频繁用来调参。频繁看 test 集再调模型，会让 test 结果失去客观性。

* signals

  npz 文件中的一维 Bz 信号数组。形状通常类似：

  ```text
  signals = [样本数, x方向采样点数]
  ```

  当前默认网格下，单个 signal 长度是 200。

* mu_maps

  npz 文件中的二维 μ map 标签。形状通常类似：

  ```text
  mu_maps = [样本数, y方向网格数, x方向网格数]
  ```

  当前默认是 `[N, 100, 200]`。

* metadata

  每个样本的参数记录，例如缺陷类型、中心位置、宽高、角度、面积、深度、提离高度、噪声水平等。第七步后还加入了多缺陷和多边形相关字段。

* metadata_keys

  metadata 每一列或每个字段的含义说明。保存它的目的是以后读 npz 文件时，不需要回头猜 metadata 字段顺序。

* defect_types

  每个样本的缺陷类型，例如 `rect`、`circle`、`ellipse`、`triangle`，以及第七步新增的 `rotated_rect`、`polygon`、`multi_defect`。

* .npz

  NumPy 的压缩数据文件格式，可以在一个文件里保存多个数组。本项目的数据集都用 `.npz` 保存，例如：

  ```text
  data/training_data_train.npz
  data/training_data_v3_complex_train.npz
  ```

## 3. 模型结构相关术语

* train_pinn.py

  训练脚本。当前负责加载 train / val 数据，构建 BzEncoder + Fourier feature + MLP 模型，支持 TV Loss、物理一致性 Loss 初版和 L-BFGS refine。

* evaluate_pinn.py

  评估脚本。负责加载 test 数据和 checkpoint，输出 MSE、MAE、IoU、Dice、area_error、center_error，并保存对比图。

* BzEncoder

  Bz 信号编码器。它把一维 Bz signal 压缩成一个 latent vector，让后续 MLP 能结合这个全局信号信息来预测每个坐标点的 μ。

* latent vector

  隐向量。可以理解为模型从 Bz signal 中提取的一组特征数字。它不直接对应某个物理量，但包含模型认为有用的信息。

* Fourier feature

  坐标特征映射方法。直接输入 `(x, y)` 有时不容易学习复杂空间变化，所以会把坐标映射成多组 `sin` 和 `cos` 特征，帮助 MLP 表达更细的空间结构。

* MLP

  多层感知机，也就是由多层全连接层组成的神经网络。本项目中 MLP 接收 `[bz_latent, coord_features]`，输出每个坐标点的 μ 值。

* batch

  一次训练同时处理的一小组样本。比如 batch size = 4 表示每一步训练同时看 4 个样本。

* epoch

  一轮训练。一个 epoch 表示模型大致看完一遍训练集。训练日志中的 `Epoch 001/020` 表示总共训练 20 轮，目前是第 1 轮。

## 4. Loss 相关术语

* Loss

  损失函数，衡量模型预测和目标之间差多少。训练的目标就是让 loss 尽量变小。

* Loss 曲线

  把训练过程中每一轮的 loss 画成图。用于观察训练是否下降、是否震荡、是否过拟合。

* MSE Loss

  Mean Squared Error，均方误差。它会对预测 μ map 和真实 μ map 的差值平方后求平均。差得越大，惩罚越重。

* TV Loss

  Total Variation Loss，总变分损失。它鼓励 μ map 更平滑，减少毛刺、孤立噪点和背景碎斑。

* Total Variation

  可以理解为图像相邻像素之间变化的总量。变化越剧烈，TV 越大。TV Loss 会让预测图不要到处剧烈跳变。

* lambda_tv

  TV Loss 的权重。总损失里通常写成：

  ```text
  total_loss = mse_loss + lambda_tv * tv_loss
  ```

  当前推荐 baseline 使用 `lambda_tv = 5e-6`。

* physics_loss

  物理一致性 Loss。第六步中，它尝试让预测 μ map 经过简化 forward model 后得到的 Bz_pred 和输入 Bz signal 匹配。目前这个模块已跑通，但不作为默认最佳模型。

* lambda_phy

  physics_loss 的权重。第六步初版使用过 `lambda_phy = 1e-4`。当前推荐 baseline 里默认 `lambda_phy = 0`，也就是不启用物理 Loss 模型作为最佳模型。

* total_loss

  总损失。第六步初版中形式是：

  ```text
  total_loss = mse_loss + lambda_tv * tv_loss + lambda_phy * physics_loss
  ```

## 5. 优化方法相关术语

* Adam

  常用神经网络优化器。它比较稳定，适合常规训练。本项目大多数训练流程以 Adam 为基础。

* L-BFGS

  一种优化方法，常用于后期精修。本项目第五步已经跑通 L-BFGS refine，但当前小子集设置下泛化指标变差，所以不作为默认推荐方案。

* refine

  精修。意思是在已有模型基础上继续做一小段优化，而不是从头训练。L-BFGS refine 就是用 L-BFGS 对已有 TV 模型做后期调整。

* parameter sweep

  参数扫描。对多个候选参数分别训练或评估，然后比较结果。例如第 5.5 步扫描了多个 `lambda_tv` 和 L-BFGS 参数组合。

## 6. 评价指标相关术语

* MSE

  Mean Squared Error，均方误差。反映预测 μ map 和真实 μ map 的整体平方误差。MSE 对大误差很敏感。

* MAE

  Mean Absolute Error，平均绝对误差。反映预测和真实之间的平均绝对差，比 MSE 对极端误差没那么敏感。

* IoU

  Intersection over Union，交并比。用于比较预测缺陷 mask 和真实缺陷 mask 的重合程度。越接近 1 越好。

* Dice

  Dice 系数，也是衡量两个 mask 重合程度的指标。越接近 1 越好。它和 IoU 都是判断缺陷区域形状是否对的重要指标。

* area_error

  缺陷面积误差。比较预测缺陷面积和真实缺陷面积差多少。越小越好。

* center_error

  缺陷中心误差。比较预测缺陷中心和真实缺陷中心距离。越小越好。

* mask 类指标

  指依赖缺陷 mask 的指标，包括 IoU、Dice、area_error、center_error。它们更关心缺陷区域的形状、面积和位置，而不是每个像素的 μ 数值误差。

## 7. 文件和目录相关术语

* checkpoints

  保存模型权重的目录。里面的 `.pt` 文件是训练得到的模型。

* checkpoint 模型

  训练过程中保存下来的模型文件，可以之后加载继续训练或评估。

* best_model.pt

  早期基线模型文件名。当前推荐最佳模型不是它，而是 `checkpoints/best_model_tv_5e-6.pt`。

* baseline

  当前用作对照的基准模型或配置。现在的 baseline 是 `checkpoints/best_model_tv_5e-6.pt`，配置为 `lambda_tv=5e-6`，默认不启用 L-BFGS 和 physics Loss。

* results

  保存实验结果的目录。已经按用途整理为多个子文件夹。

* summaries

  `results/summaries/`，保存实验总结文本，例如参数扫描总结、物理 Loss 对比总结。

* metrics

  `results/metrics/`，保存评估指标文件，例如 `evaluation_metrics.txt`、`physics_loss_comparison.csv`。

* loss_curves

  `results/loss_curves/`，保存训练 loss 曲线图。

* previews

  `results/previews/`，保存预测图、重建图、数据可视化检查图。

* sweeps

  `results/sweeps/`，保存参数扫描表，例如 `tv_lambda_sweep.csv` 和 `lbfgs_sweep.csv`。

* archive

  `results/archive/`，保存暂时不属于上述类别的日志或辅助文件，例如训练日志、`physics_loss_log.csv`。

## 10. 当前项目中的重要判断原则

* MSE 下降但 IoU / Dice 变差

  说明模型整体像素误差变小了，但缺陷区域的形状重合变差了。对本项目来说，缺陷形状反演很重要，所以不能只看 MSE。第六步物理 Loss 初版就是这种情况：MSE 略好，但 IoU 和 Dice 变差，因此没有成为默认最佳模型。

* MAE 有改善但 mask 类指标没改善

  说明平均像素误差变小了，但缺陷区域的位置、面积或形状没有更准。第四步 TV Loss 初版曾出现类似情况，因此后续才做了 `lambda_tv` 参数扫描。

* L-BFGS 泛化指标变差

  说明 L-BFGS refine 可能把小子集拟合得更好，但在 val/test 上效果不稳定。当前项目中 L-BFGS 流程已跑通，但不作为默认推荐方案。

* test 集不要频繁调参

  test 集应该像最终考试一样使用。如果每次调参都看 test 结果，就相当于提前看答案，会导致最终指标不客观。本项目主要用 val 集选择参数，test 集用于阶段性最终评估。

* baseline 不应该随便覆盖

  baseline 是项目当前可信的对照点。即使某个新模型某个指标变好，也要看整体指标，特别是 mask 类指标。当前 baseline 仍是 `checkpoints/best_model_tv_5e-6.pt`。

* 新数据集不应该覆盖旧数据集

  旧数据集对应已有实验和模型指标。如果覆盖旧数据，之前的结果就很难复现。第七步复杂缺陷数据使用 `training_data_v3_complex_*.npz`，就是为了和旧的 `training_data_*.npz` 区分。
