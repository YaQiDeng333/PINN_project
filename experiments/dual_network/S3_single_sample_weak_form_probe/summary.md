# S3_single_sample_weak_form_probe

## Purpose

Run the first recordable single-sample weak-form probe for the dual-network branch. This experiment uses a small `.npz` dataset generated by `data_generator_v2.py` and runs the compact-support weak-form path in `minimal_dual_single_sample_loop.py`.

This is still a branch probe, not formal training and not evidence that the method outperforms the main supervised baseline.

## Data Generation

Command:

```powershell
python data_generator_v2.py --train-samples 1 --val-samples 0 --test-samples 0 --grid-x 20 --grid-y 10 --output-dir experiments/dual_network/S3_single_sample_weak_form_probe/data --seed 123
```

Generated files:

- `experiments/dual_network/S3_single_sample_weak_form_probe/data/training_data_train.npz`
- `experiments/dual_network/S3_single_sample_weak_form_probe/data/training_data_val.npz`
- `experiments/dual_network/S3_single_sample_weak_form_probe/data/training_data_test.npz`

The single-sample loop used:

```text
experiments/dual_network/S3_single_sample_weak_form_probe/data/training_data_train.npz
```

## NPZ Fields

For `training_data_train.npz`:

```text
signals:      shape=(1, 20), dtype=float32
mu_maps:      shape=(1, 10, 20), dtype=float32
defect_types: shape=(1,), dtype=<U16
metadata:     shape=(1,), structured dtype with defect and generator parameters
x:            shape=(20,), dtype=float32
y:            shape=(10,), dtype=float32
```

Required branch fields were present:

- `signals`
- `mu_maps`
- `x`
- `y`

No `coords` field was required because `dual_network_data_utils.py` can construct coords from `x/y`.

## Loop Configuration

Command:

```powershell
python minimal_dual_single_sample_loop.py --npz-path experiments/dual_network/S3_single_sample_weak_form_probe/data/training_data_train.npz --sample-index 0 --outer-steps 10 --phi-steps 20 --mu-steps 20
```

Configuration:

- `sample_index = 0`
- `outer_steps = 10`
- `phi_steps = 20`
- `mu_steps = 20`
- compact-support test centers: `[-5, 5]`, `[0, 5]`, `[5, 5]`
- `test_radius = 5.0`
- `test_grads_shape = (3, 200, 2)`

Full stdout was saved to:

```text
experiments/dual_network/S3_single_sample_weak_form_probe/run_stdout.txt
```

## Interpretation

This confirms that the branch can connect data generated by `data_generator_v2.py` to the single-sample compact-support weak-form loop and complete a finite optimization probe.

The result is only a code-path and stability probe. It does not validate defect reconstruction quality, and it does not establish physical consistency of the generated `signals` with a strict PDE forward model.

## Current Limits

- The generated `signals` still come from the current analytic MFL approximation.
- Test centers and radius are manually fixed.
- No quadrature weights are used.
- No model checkpoint or image output is saved.
- No comparison against the main supervised baseline is made.
