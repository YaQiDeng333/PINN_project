# S44 200x100 后处理/阈值诊断实验

## 1. 数据来源

- 数据来源：`experiments/dual_network/S42_200x100_fresh_area3_bce7_validation/area3_bce7/`。
- 样本数量：100 个 200x100 样本。
- 本轮只读取每个样本的 `final_mu_pred.npy` 和 `final_mu_label.npy`，未重新训练。
- `label_mask` 定义为 `mu_label < 500`。

## 2. raw_threshold_500 结果

- 平均 IoU：`8.047192e-01`。
- 平均 `defect_area_pred`：`1.213090e+03`。
- 平均绝对 area error：`3.355900e+02`。
- 这是 S42 `area3_bce7` 的默认 mask 结果，对应 `pred_mask = mu_pred < 500`。

## 3. threshold sweep 最佳结果

- 平均 IoU 最佳阈值：`500`，平均 IoU `8.047192e-01`。
- 面积误差最小阈值：`500`，平均绝对 area error `3.355900e+02`。
- 较低阈值可以明显降低预测面积，但 IoU 同时显著下降：`threshold=400` 的平均 IoU 为 `4.880164e-01`，平均面积为 `8.003000e+02`；`threshold=300` 的平均 IoU 为 `2.143824e-01`，平均面积为 `5.481700e+02`。
- 当前判断：低阈值确实压低 false-positive 面积，但会损伤召回和整体 IoU；直接阈值扫描没有超过 S42 默认的 `500`。

## 4. largest connected component 结果

- LCC 平均 IoU 最佳阈值：`500`，平均 IoU `8.058842e-01`。
- LCC 平均 `defect_area_pred`：`1.181090e+03`。
- LCC 平均绝对 area error：`3.052100e+02`。
- 相比 raw threshold，LCC 只带来很小的平均 IoU 提升，并略微降低预测面积。
- 风险：如果真实缺陷包含多连通区域、边界细节或细长复杂形状，最大连通域过滤可能删除有效召回区域，因此它更适合作为诊断或可选后处理，而不是直接替代训练目标。

## 5. oracle area top-k 结果

- oracle top-k 平均 IoU：`7.788534e-01`。
- oracle top-k 平均 `defect_area_pred`：`8.829200e+02`。
- oracle top-k 平均绝对 area error：`0.000000e+00`。
- 该方法使用真实 `label_area`，只作为诊断上界，不是可部署方法。
- oracle top-k 低于 raw threshold，说明单纯把面积校准到真实面积并不能提升结果；`mu_pred` 的空间排序本身仍存在局部错误。

## 7. 当前判断

- 当前 false positives 不是单纯阈值问题：阈值低于 500 虽然降低面积，但 IoU 明显下降。
- 当前 false positives 也不是单纯面积校准问题：oracle top-k 使用真实面积后平均 IoU 仍低于 raw threshold。
- 最大连通域过滤可以略微降低面积并带来非常小的平均 IoU 提升，但收益有限，且可能损伤多缺陷或复杂形状样本。
- 主要瓶颈更接近 `mu_pred` 空间排序和局部形状信号不足，而不是简单后处理可以完全解决的问题。
- 下一步如果继续 200x100，建议把 LCC 当作可选后处理 ablation 在 fresh 数据上验证；更核心的改进应转向结构性 false-positive 控制、弱样本形状分析或损失设计。
- 边界不变：S42/S44 仍属于半监督 BCE mask prior 诊断上界，不是无监督 weak-form 成功。
